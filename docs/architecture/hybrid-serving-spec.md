# Hybrid Serving Architecture Spec (v1)

## 1. Component Model

The hybrid architecture transitions from a single PostGIS/GeoPackage backend to a multi-tiered serving strategy using **DuckDB** for on-demand high-churn river-flow tiles and **PMTiles** for stable/base vector layers.

### 1.1 Components
- **CloudFront**: Edge cache and SSL termination. Routes requests to the Application Load Balancer (ALB).
- **ALB (Application Load Balancer)**: Performs path-based routing.
  - Path `/riverlines/*` -> **DuckDB-Tile-Server** (New)
  - Path `/tiles/*` -> **PMTiles-Serving-Path** (New Path for stable layers)
- **DuckDB-Tile-Server (New)**: High-performance server reading GeoParquet from S3/EFS using DuckDB. Serves the high-churn `riverlines` layer on-demand.
- **PMTiles-Serving-Path**: S3-native or Lambda-based serving for stable/base vector tiles in PMTiles format.
- **Data Store (S3/EFS)**: 
  - GeoParquet files for the `riverlines` layer (High-Churn).
  - PMTiles files for base vector layers (Stable).

## 2. Canonical Routing Matrix and Precedence

Strict routing rules ensure compatibility with existing client contracts.

| External Path | Internal Route | Target Component | Layer Class |
| :--- | :--- | :--- | :--- |
| `/riverlines/{z}/{x}/{y}.pbf` | `/serve/riverlines/{z}/{x}/{y}.pbf` | DuckDB-Tile-Server | High-Churn (DuckDB/Parquet) |
| `/tiles/{z}/{x}/{y}.pbf` | `/tiles/{z}/{x}/{y}.pbf` | PMTiles-Serving-Path | Stable (PMTiles) |

**Precedence Rule**: ALB MUST prioritize specific paths like `/riverlines/*` before falling back to generic `/tiles/*` routing.

## 3. Cache Policy by Layer Class

| Layer Class | Cache Strategy | TTL (CloudFront) | Revalidation |
| :--- | :--- | :--- | :--- |
| **Stable** (PMTiles) | Long-term immutable | 30 Days | On file update |
| **High-Churn** (DuckDB) | Short-term | 5 Minutes | Every request (ETag) |

## 4. Failure Modes and Fallback Behavior

| Component | Failure Mode | Behavior | Fallback |
| :--- | :--- | :--- | :--- |
| DuckDB-Tile-Server | Service Down / 5xx | ALB returns 502/503 | None |
| S3/EFS (GeoParquet) | File Missing | DuckDB returns 404 | Empty PBF tile |
| PMTiles Path | S3 Object Missing | 404 Not Found | Empty PBF tile |

## 6. Phase-1 Scope Boundaries

- **IN SCOPE**: Hybrid routing at ALB level. New DuckDB-based serving for `/riverlines`. PMTiles path for `/tiles`.
- **OUT OF SCOPE**: Redesigning raster tile pipelines. Migrating all PostGIS generic queries to DuckDB. Full t-rex decommissioning.
